# Отладка белого экрана (Frontend)

Если при запуске фронтенда сайт загружается, но на экране ничего не видно (белый экран), выполните следующие шаги:

1. Проверьте порт
   - По умолчанию `start-dev.bat` указывает `http://localhost:3000`, но Vite может автоматически переключиться на другой порт (например, 3001) если 3000 занят.
   - В терминале, где запускаете `npm run dev` для фронтенда, Vite выведет адрес для локального доступа — используйте именно его.

2. Откройте инструменты разработчика в браузере (F12) и обратите внимание на:
   - Консоль: ошибки runtime (Uncaught TypeError, ReferenceError и т. п.)
   - Network: 404 или неудачные запросы (особенно `/src/main.tsx` или `/index.html`)

3. Частые причины белого экрана
   - Неправильный порт: сервис слушает не на 3000, а на 3001 — откройте указанный терминалом адрес.
   - JS-исключение при первом рендере — ErrorBoundary (сейчас добавлен) покажет сообщение об ошибке.
   - Внешние запросы к бэкенду приводят к ошибкам CORS/401, но UI всё равно должен рендериться — проверьте консоль.

4. Как принудительно освободить порт 3000 (Windows PowerShell)
   - Найти процесс: `netstat -ano | Select-String 3000`
   - Проверить PID и завершить: `Stop-Process -Id <PID>`

5. Что делать дальше
   - Если консоль сообщает точную ошибку — пришлите её сюда (скрин/логи). Это ускорит решение.
   - Если ошибка видна в ErrorBoundary — пришлите текст ошибки и стек.

6. Временное решение
   - Если Vite автоматически использует порт 3001 — откройте `http://localhost:3001/`.
